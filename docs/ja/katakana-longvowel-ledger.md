# カタカナ長音符（ー）表記の決定台帳

日本語訳における外来語末尾の長音符「ー」の有無を統一するための決定記録。
`docs/ja/translation-glossary.md` の補助資料（エージェント用内部資料、読者には出ない）。

## 規則（3層）

判定軸は文字数でも音訳/意訳でもなく、**対象読者（プログラマー）にとっての
定着度・明快さ・誤解リスク**。

1. **スコープゲート** ── 長音を落とす判断の対象は英語語尾 **-er / -or / -ar /
   -ure** に由来する `ー` だけ。-ow（ワークフロー）・-ough（ウォークスルー）・
   -y/-ee など他由来の `ー` と語中の `ー` は**常に伸ばす**（候補外）。
2. **デフォルトは伸ばす（KEEP ー）**。安全側。
3. **落とす（DROP）のは「短縮形そのものが工学慣用として定着している語」だけ**
   （パラメータ・ハンドラ・クラスタ等、数十年のJIS/IT慣用がある語）。
4. **-ry / -ly / -ory ゾーンは全部伸ばす**（エントリー・サマリー等。どちらでも
   許容なので伸ばす方が安全）。

複合語は **run（連続カナ列）内の全要素**に規則を適用する（例：
`ポインターパラメーター` → `ポインタパラメータ`、`レシーバクラスター` →
`レシーバークラスタ`）。コードフェンス・インラインコード内は対象外。

## DROP（ー → なし）── 短縮形が工学定着

パラメータ・ハンドラ・クラスタ・ポインタ・フィルタ・コンパイラ・コンストラクタ・
ジェネレータ・アクセサ・セレクタ・プロセッサ・セパレータ・コンテナ・
インタープリタ・スケジューラ・リゾルバ・シリアライザ・コンビネータ・
フォーマッタ・アキュムレータ・スキャナ・ディスクリプタ・エディタ・パーサ・
プロバイダ・ドライバ・カウンタ・アダプタ・リファクタ・イテレータ
／-ure：クロージャ・フィクスチャ・プロシージャ・インフラストラクチャ

（イテレータ・コントローラーは後追いの片形統一。コントローラは多数派の
コントローラーへ伸ばす側で統一＝KEEP。iterator は短縮形定着で DROP。）

## KEEP（なし → ー）── 短縮形が未定着・一般語・行為者名詞、および -ry ゾーン

プロデューサー・コンシューマー・ウォーカー・メンバー・ファミリー・レイヤー・
パブリッシャー・ディスパッチャー・レシーバー・サーバー・レター・ラッパー・
ミューテーター・トレーラー・バリデーター・ダーティー
／要確認だった7語も全部伸ばす：トリガー・アナライザー・ビルダー・メンテナー・
プレースホルダー・バインダー・マネージャー
／-ry/-ly/-ory：エントリー・サマリー・カテゴリー・ファクトリー・ディスカバリー・
アドバイザリー

判定の手触り（KEEP 側）：
- 短縮形が不自然（ウォーカ・フレーバ）。
- 短縮形が既存語と衝突（マッチャ ≈ 抹茶）。
- 一般日常語で `ー` が強く慣用（メンバー・ファミリー・サーバー）。

## 漢訳（カタカナをやめる・別軸）

- `evaluator` → **評価器**（`adr/16-macro-expansion`。初出ルビ
  `<ruby>評価器<rt>evaluator</rt></ruby>`、以降素の評価器。「Lisp風マクロ評価器」）。
- `language server`（ランゲージサーバー）→ **言語サーバー**（ランゲージ→言語、
  server は KEEP のサーバー）。
- `recognizer`（レコグナイザー）→ **認識器**（定義ページ `adr/30` 初出パーレン
  併記。コード `BindingRecognizer` は据え置き）。
- `comparator`（コンパレーター）→ **比較器**（定義ページ
  `design/…cache-slice-taxonomy` 初出パーレン併記。CS標準訳）。
- `input adapter`（インプットアダプタ）→ **入力アダプタ**（「Sorbet入力アダプタ」）。
- `contributor`（コントリビューター）→ **貢献者**（「マルチコントリビューター」→
  「複数貢献者」。別語の contribution＝コントリビューションは対象外）。
- `early return`（アーリーリターン）→ **早期リターン**（narrowing はナローイング維持。
  adr/24・notes の既存表記に統一）。
