# 日本語翻訳メモリ（用語集・表記ルール）

このファイルはrigor.typedduck.failの日本語翻訳作業で決定した訳語と表記ルールを管理します。
新しい訳語が決まったらここに追記してください。

## 訳語一覧

| 原語 | 訳語 | 初出時の注記形式 | 備考 |
|---|---|---|---|
| gradual | グラデュアル | 単体の初出: グラデュアル（漸進的、段階的）。「グラデュアル一貫性」のように複合語になる場合は初出で（gradual consistency）と英語を添え、リストの直後に「「グラデュアル」は日本語で「漸進的」「段階的」とも呼ばれます。」を挿入 | 型理論の "gradual typing" 文脈 |
| carrier | キャリア | — | カタカナのまま（訳さない） |
| nominal type | 名前的型 | 見出しはシンプルに「名前的型」、本文初出で「名前的型（nominal type）は公称型とも呼ばれます」と補足 | TAPLの訳語に準拠 |

## 用語紹介の表記ルール

専門用語を本文中で**初めて紹介するとき**は、英語原語と他の一般的な訳語を全角括弧で添える。

形式: `日本語訳（英語原語、別訳語とも）`

例:
- 名前的型（nominal type、公称型とも）
- グラデュアル（漸進的、段階的）

2回目以降は括弧注記なしで日本語訳のみ使う。

## 変更しないもの

- RFC 2119 キーワード（MUST, SHOULD, MAY, MUST NOT, SHOULD NOT）は英語大文字のまま
- コードブロック・型名・識別子は原文のまま
- `RBS::Extended`, `Dynamic[T]`, `~T`, `T - U` などのRigor型表記は原文のまま
