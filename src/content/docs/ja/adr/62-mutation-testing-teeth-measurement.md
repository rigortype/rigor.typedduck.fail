ステータス: **Accepted — ハーネスと最初の修正は2026-06-13に着地。方法論とその裁定の<ruby>規律<rp>（</rp><rt>discipline</rt><rp>）</rp></ruby>は有効。残りのバックログは需要ゲートされている**。開発専用のミューテーションハーネス（`tool/mutation/`）は、解析対象コードを型から見える形で壊し、*生き残った*ミュータント（mutant）を偽陰性（false negative）の候補として読み取る — Rigorの反偽陽性（anti-false-positive）の規律の双対である。`lib/rigor`へのスイープからランク付けされたバックログが生まれ、そこから2件の修正が着地した（ユニオンレシーバーの未定義メソッドの歯（teeth）、RBSクラスエイリアスの解決）。1件の候補は*意図的な沈黙*と裁定されて先送りされ、残りは下記に記録する。

根拠: [`docs/notes/20260613-mutation-teeth-harness.md`](../../notes/20260613-mutation-teeth-harness/)（メカニズム、スイープのテーブル、クラスタごとのトリアージ）。

## コンテキスト

Rigorの開発規律全体は**反偽陽性**である。「プログラムは動く」が最悪ケースの静的な読みに優先し（[`feedback_false_positive_discipline`]）、コーパスのバイト単位同一性ゲートは*振る舞いを変えたか*を計測する。**双対軸** — *そもそも歯（teeth）を持っているのか*: コードを壊してもRigorが文句を言わないのはどこか（偽陰性／盲点）— を体系的に計測するものは何もなかった。「歯を検証済み（teeth verified）」はADR-43／ADR-24をまたいで逸話的に登場するが、計測されリグレッション追跡可能な量になったことは一度もない。これがそのギャップであり、凍結前の窓（v0.1.19以降）こそそれを閉じる価値のあるときである。診断の*語彙*がまさに凍結されようとしており（ADR-50）、その背後にあるルールが噛みつくという証拠が欲しいからである。

## 決定

偽陰性を計測するための恒常的な実践として、**解析器のミューテーションテスト（mutation testing）**を採用する。クリーンなファイルに型から見えるミューテーションを注入し、ミューテートされたバイト列に対して解析を再実行し、*生き残った*ミュータント（新しい診断が出ない）を偽陰性の候補として扱う。出力をどう読むかを2つの基準が支配する — これらは再利用可能なルールであって、ツールそのものではない:

- **基準A — 生存者がシグナルになるのは、Rigorが具体的な型を保持している箇所に限る**。型チェッカーはバグの部分集合しか見ないので、ほとんどのミューテーションは型不変な*等価ミュータント（equivalent mutant）*であり、それらの生存は**正しい**（偽陽性の規律が働いている）。したがって素のキルレート（kill-rate）はノイズである。**型認識アンカーフィルタ（type-aware anchor filter）**（ミューテーションが契約（contract）を侵害し得るレシーバーが非`Dynamic`に型付けされる箇所だけをミューテートする）こそが、この指標に意味を持たせる。これは装飾ではなく荷重を担う部分であり — そしてこれがハーネスがインプロセスである理由である（インプロセスのツールだけがエンジンに自身の`type_of`を問い合わせられる）。
- **基準B — 仮定せず、裁定せよ**。生存者は*候補*であってバグではない。各々を仕様／既存の決定に照らして裁定する。一部は**意図的な沈黙**である（下記N3の`T | nil`のケース）。修正が着地するのは、健全性が**完全**であり`make verify`がクリーンに保たれる（lib＋プラグインをまたいで新規発火なし）箇所に歯を加えるときに限る。これはADR-57が用いたのと同じ、クラスごとに裁定するプロトコルである。

## ワーキングデシジョン

- **WD1 — Prismネイティブ・インプロセスで自前に作る。`mbj/mutant`は却下**。意味のある指標には型認識のサイト選択（基準A）が必要で、それにはエンジン自身の`type_of`が要る — それができるのはインプロセスのツールだけである。`mutant`はwhitequark-`parser`ベースでもある（RigorはPrismであり、ソーステキストの境界でのみ無害）。そのプロセスモデルはウォームなインプロセスループと衝突し、その演算子セットは振る舞いテストスイート向けにチューンされている（大半は型不変 → 型チェッカーにとってはノイズ）。ミューテータはPrismのバイトオフセットでソースをスプライス（splice）する — アンパーサ不要。解析器が再パースする。`tool/mutation/mutate.rb`。
- **WD2 — ウォームループがコーパス規模のスイープを実用にする**。 `LanguageServer::ProjectContext`がRBS環境＋プロジェクト全体の`ProjectScan`を一度だけ構築し、各ミュータントは`Runner.new(environment:, prebuilt:)`＋`#run_source`（メモリ内オーバーレイ）でそれらを再利用する。`prebuilt:`は`run_result_cacheable?`を強制的にfalseにするので、*ディスク*ファイルをダイジェストするラン結果キャッシュ（run-result cache）がバイパスされ、ミュータントが古いクリーンなヒットを返されることは決してない。コールド約400 ms、その後はミュータントあたり約6〜12 ms。
- **WD3 — バックログとしてのスイープ**。 `mutate.rb sweep <paths…>`は1つのウォームセッション上でコーパスを走らせ、生存者を`(operator, receiver type)`でクラスタリングしてカウント順にランク付けする。`--json`は同じものをエージェント向けに出力し（ADR-61の流儀）、自己改善ループ（スイープ → エージェント修正 → 再計測キル）を閉じる。デフォルトの演算子は`arity_extra`を除外する（可変長／オプショナルなメソッドに引数を追加するのは等価ミュータント — ノイズ）。フィルタは`Dynamic`／`Top`／`Bot`のいずれかのアームを持つユニオンを非具体的（non-concrete）として扱う。
- **WD4 — 開発専用。ADR-50の凍結サーフェスの外**。ハーネスはCLIコマンドではなく、診断契約に何も寄与しない。エンジン作業を駆動するものであり、出荷物ではない。
- **WD5 — 着地済みの修正（ループが機能している証拠）**。
  - *ユニオンレシーバーの未定義メソッドの歯。* `call.undefined-method`が、メソッドが**すべての**非nilアームで欠けているときにユニオンレシーバーに対して発火するようになった（`A | B`が`m`に応答するのは両方が応答するとき**かつそのときに限る**）。構造上、偽陽性に安全である: `union_undefined_method_diagnostic`（`check_rules.rb`）は寛容な`method_present_anywhere?`を再利用し（任意のDynamic／未知／ソース宣言のアーム ⇒「存在する」⇒ 発火しない）、加えて開いた（ADR-26）／合成された／シングルトン／モジュールミックスインのいずれかのアームで脱出するアームガードを持つ。純粋な非nilユニオンに限る。
  - *RBSクラスエイリアスの解決。* `class Mutex = Thread::Mutex`は`class_alias_decls`にのみ存在したため、`class_known?`はそれを報告したものの、定義ビルダー（`class_decls`でガードする）はそのメソッドを列挙できず — すべてのエイリアスクラスが「既知だがメソッドなし」のまま残っていた。`RbsLoader#canonical_module_name`がガードの前にエイリアスをそのターゲットへ正規化し、ディスパッチ*と*`Mutex`およびあらゆる`X = Y`の存在チェックを修正する。

## 却下／先送りした代替案

| 代替案 | 裁定 |
| --- | --- |
| エンジンとしての`mbj/mutant` | **却下** — 型認識のサイト選択にはインプロセスの`type_of`が必要。パーサの不一致＋プロセスモデル＋振る舞いスイート向け演算子（WD1）。 |
| デフォルト演算子セットに`arity_extra`を含める | **オプトインに降格** — ほとんどのRubyメソッドは追加の引数を受け取る（スプラット／オプショナル）ため、等価ミュータントのノイズ工場である。シグネチャアリティ（arity）ガードがあればデフォルトに値するようになる（フォローアップ）。 |
| **nilable**なユニオン（`String?`）で発火する | **検討＋却下 — N3の沈黙を維持**。 N3の決定（`safe_navigation_undefined_method_spec.rb`）は意図的に`T \| nil`を沈黙させたままにする。バンドルされたアームでナローイングした候補を13プロジェクト（ActiveSupport中心＋素のもの）で走らせた結果: **真のnilableユニオン発火はゼロ**（歯の増分は約0）である一方、セルフチェックは本物の特異性喪失の偽陽性を表面化させた（`plugin_class : Class`が`.manifest`を持つ`Plugin`サブクラスを保持する）。増分ほぼ0＋偽陽性のしやすさ＋意図的な決定を覆すコスト ⇒ 割に合わない。それでもこの調査は、収穫した2つの偽陽性ガードで*出荷済みの*スライス1を堅牢化した: 汎用`Class`／`Module`アームガードと、**異クラスガード（distinct-class guard）**（真にマルチクラスのユニオンに限って発火し、`Hash[K1] \| Hash[K2]`のシェイプ結合には発火しない — これがmailの`compose_codepoints`での本物の外部スライス1偽陽性を修正した）である。 |
| `argument-type-mismatch`の低歯クラスタ | **先送り** — 混在している: `h[nil]`は正しい（任意のキー）が`Integer#>=(nil)`は本物の取りこぼしである。一律修正ではなくサイトごとの裁定が必要。 |
| `Type::*`キャリアの欠落メソッドRBS（セルフドッグフード） | **先送り** — シグネチャカバレッジの作業。セルフチェックの歯を改善するが、ユーザー向けの緊急性はない。 |
| 広域ファズ（broad-fuzz）モード（クラッシュ／ハング／健全性の検出） | **着地 — クリーン**。 `mutate.rb fuzz <paths…>`はウォームループを攻撃的なフィルタなしのミューテーション（あらゆる演算子、あらゆるサイト）で走らせ、解析器をクラッシュさせる（`internal analyzer error:` — Runner自身のレスキュー）、ハングさせる（ミュータントごとのタイムアウト）、または — `--repeat`付きで — 非決定的な診断を返す（キャッシュのバイト単位同一性契約を壊し得る）ミュータントを報告する。健全性チェックは*非決定性*（2回走らせて比較）であって、「ベースライン診断を除去／反転させる」ことではない（任意のミューテーションに対しては定義が不明瞭）。初回ラン: `lib/rigor`全体に対して2,706ミュータント、**クラッシュ／ハングゼロ** — 解析器は自身のツリーの任意の型から見えるミューテーションに対して堅牢である。 |
| ユーザー向けの「型保護カバレッジ（type-protection coverage）」レポート | **提案／需要ゲート** — ファイル／メソッドごとに、Rigorがキルする型関連ミューテーションの割合を報告する`coverage`コマンドの兄弟（「型を追加すれば本物の保護が買える箇所」）。常に有効性／どこに型を追加すべきかとして枠付けし、**決して**素の生存率としては枠付けしない（さもなくば基準Aの罠が動作中のコードを怯えさせる）。まず内部ツール＋演算子セット＋性能のストーリーを成熟させること。 |

## 帰結

- **正** — 歯が計測されリグレッション追跡可能な量になる。スイープは手で見つけたギャップに頼る代わりにランク付けされたエンジンのバックログを*生成*する。`--json`の経路はそれをエージェントが処理可能な自己改善ループにする。本物の偽陰性が2件すでに閉じられ、各々`make verify`クリーンである。
- **負** — 解釈の負担: 素のキル率（%）は誤解を招く（アリティノイズで水増しされ、引数チャネルで目減りする）ので、この実践はフィルタ＋裁定の規律が適用されることに依存する。スイープはCI時間のものであり、対話的ではない。ハーネスは開発のメンテナンスサーフェスである（エンコーディング、ミュータントの妥当性、演算子のカバレッジ）。
- **持ち越し** — 上記のバックログ行が需要ゲートされたキューである。根拠ノートが生きたトラッカーである。

## 他のADRとの関係

- **偽陽性の規律の双対**。コーパスのバイト単位同一性ゲート（「振る舞いを変えたか」）を補完し、`rigor-regression-sweep`スキル（偽陽性／表面化した診断のドリフトを追跡する）の偽陰性の兄弟である。
- **ADR-23／ADR-61** — `--json`バックログは、それらのADRが確立した構造化されエージェントが消費可能な形を再利用する。
- **ADR-26** — ユニオンの歯ルールのアームガードは、開いたレシーバーのサーフェスを尊重する。
- **ADR-57** — 同じクラスごとに裁定する規律（基準B）。
- **ADR-50** — ハーネスは意図的に凍結された公開サーフェスの外に置かれる（WD4）。
- **ADR-43／ADR-24** — それらのADRが逸話的に主張する「歯を検証済み」という主張を、計測可能な量にする。
