実装に入ったスライス（slice）は**基盤**である: `[class, method, kind]`をキーとする呼び出しサイトの引数ユニオン収集器（`Inference::ParameterInferenceCollector`）、`param_inferred_types`という`DiscoveryIndex`の副テーブル、そして`build_method_entry_scope`での消費（宣言されていないパラメータには推論された型がシードされ、RBSで宣言されたパラメータが優先される）だ。これは**上限付き不動点**（WD5）として動く: 各ラウンドで前ラウンドの推論済みパラメータをシードしてプロジェクトを再型付けするので、*別の*パラメータを渡されたパラメータはラウンドごとに1ホップ先まで型付けされる（上限3、`BodyFixpoint`の慣習、収束時には早期停止。ラウンド1だけならシングルレベル（single-level）パスにあたる）。引数がまだ型付けされていないパラメータである呼び出しサイトは、*そのラウンドでは*そのパラメータを毒する（WD4）が、自分の引数が解決すれば後のラウンドで型付けされうる。配線は**`coverage --protection`のみ**だ: `check`ウォークはテーブルを空のまま残すので、その診断はバイト単位で同一であり、WD1の「パラメータ境界では決して発火しない」は*構成上*成り立つ（推論された型は本体ローカルであってRBS契約（contract）ではなく、境界ルールはRBSを参照する）。

**計測された留保（2026-06-16の検証。蒸し返さないこと）:**看板ケースとして挙げられた2つのADRは、不動点を使ってさえ*動かない*。parserの`unary_num`パラメータは生成された`.y`の値スタックコード（`val[0]`、解析されるRubyではない）から供給される。faradayの`match(env)`は`env`を引数に呼ばれるが、その`env`自体が`call(env)`のパラメータであり、`call`のエントリーは動的なミドルウェアディスパッチで到達される（だから不動点はその根をシードできない）。本パスがメトリックを動かすのは、ユーザーメソッドが（推移的に）具体的に型付けされた引数で呼ばれている箇所だ。**計測値**（`coverage --protection`、`lib`）: faraday 0.2129 → 0.2402（+29保護サイト。うち+6はシングルレベルに対する不動点による）、haml 0.3188 → 0.3842（+105サイト。うち+6が不動点による）── hamlの`compile(node)`スタイルのコンパイラチェーン（構築されたASTノードで呼ばれるメソッド群）こそが好適例であり、挙げられた`env` / `numeric`クラスタはそうではない。`check`ウォークへの配線（およびそれに伴うWD1の本体内（in-body）由来（provenance）*マーク*）が残るフォローアップであり、下記のコスト議論に沿って予算ゲート付きである。

## コンテキスト

RBSシグネチャを持たないメソッド／コンストラクタのパラメータは`untyped`型になる（漸進的なエントリポイント。`method_dispatcher.rb`の「鍵となるパラメータは`untyped`のまま残される──既定値だ」）。ブロックパラメータはディスパッチから推論されるが、**`def`パラメータは推論されない**──呼び出しサイトからも本体内（in-body）の使用からもだ。だから`def initialize(line); @line = line; end`は`@line`を`Dynamic`にし、下流のあらゆる`@line.<method>`が未保護になる──これは[ADR-58](../58-ivar-field-typing/)とは異なる。ADR-58のインスタンス変数型付けは具体的書き込みフィールドについては*すでに実現済み*だが、**パラメータ由来**のインスタンス変数には手を出せない（確認済み: 具体的書き込み`@line = Line.new`は`Line`を読み取って保護するが、パラメータ由来の`@line = line`は`Dynamic`を読み取る）。

アルゴリズム調査が**M3をexcusedした**のは、そのコーパスが呼び出しサイトを持たないリーフスクリプトだったからだ──推論の*種*がなかった。保護パイロットはコスト／ベネフィットを変える: 実アプリケーションでは呼び出しサイトが存在し（faradayの`Connection`/`Options`、hamlのコンパイラチェーン）、M3は未保護クラスタのうち残る筆頭であり、保護カバレッジこそがそれを閉じる価値を測るレンズである。常につきまとう緊張は[ADR-5](../5-robustness-principle/)だ: パラメータは決定により**寛容（lenient）**に保たれる（戻り値は厳格、パラメータは寛容）ので、いかなるパラメータ型付けも誤った型を渡す呼び出し元を診断に変えてはならない。

## 決定

パラメータ型を2つのソースから、段階的に、ただ1つの基幹となる基準のもとで推論する。

> **基準（ロバストネスとの整合）:**推論されたパラメータ型は**精度加算的（precision-additive）であるに限る**。それは*下流の*推論を鋭くする──パラメータが格納されるインスタンス変数、パラメータが成るレシーバー、保護メトリック、定数畳み込み──が、呼び出し元の引数を拒否するためには**決して**参照されない。パラメータ境界は寛容なままだ（ADR-5）: 想定外の型を渡す呼び出しは診断にならない。RBSで*宣言された*パラメータは常に推論されたものに優先する。推論は保護／畳み込みに供されるのであって、パラメータ境界の診断には供されない。

これが本提案を「型付きパラメータ」体制から分かつものであり、ロバストネス原則を破ることなく進められる理由である。

## ワーキングデシジョン

- **WD1 — 精度加算的に限る契約（robustnessの床。交渉の余地なし）。構成上の実装（シングルレベルスライス）**。推論されたパラメータ型は、決してパラメータ境界の`argument-type-mismatch` / アリティ（arity）発火へエスカレートできない。実装に入ったスライスでは、これは*構造的に*成り立つ: 推論された型は（`build_method_entry_scope`を介して）メソッド本体ローカルとしてのみ格納され、RBSメソッド定義へ注入されることはなく、境界ルールはRBSで宣言されたメソッドにしか発火しない──推論されたパラメータを持つメソッドはRBSシグネチャを持たないので、境界ルールはそれをスキップする。明示的な「推論であって宣言ではない」という由来（provenance）*マーク*（[ADR-58](../58-ivar-field-typing/)のWD1パターンの再利用）が要るのは、推論が`check`ウォークに供給されるようになったときに*本体内*の診断を防ぐためだけであり、そのフォローアップとともに実装される。
- **WD2 — 本体内使用の下界（最も安価。リーフスクリプトにも効く）**。メソッド本体でのパラメータの呼び出し（`arr.delete_at`、`arr.length`）から、**構造的下界（structural lower bound）**（応答可能集合／インターフェース）を導出し、それで`arr.<method>`の保護を駆動させる。呼び出しサイトは不要なので、調査がexcusedしたリーフスクリプトコーパスにも効く──ただし得られるのはダック／構造的な下界であって、名前的型ではない。
- **WD3 — 呼び出しサイトユニオン（TypeProf流。アプリに対する本当のてこ）。実装済み**。パラメータの推論型 = 解決された呼び出しサイトの引数型のユニオン（解決された呼び出しサイトが1つ以上必要）。`ParameterInferenceCollector`は、クロスファイルの探索インデックスを介して呼び出しをそのユーザー`def`へ解決し、位置引数を型付けし、パラメータごとにユニオンをとり、単純でないパラメータシェイプ／アリティ不一致／スプラット呼び出しをスキップする。これは全プログラム的であり、Rigorのファイル単位モデル（[ADR-46](../46-incremental-dependency-graph/)）と緊張関係にあるので、`coverage --protection`コマンドの内側でのみ動き（`check`ウォークでは動かない）、**予算ゲート付き**である（パラメータごとのユニオン上限`MAX_CALL_SITE_TYPES`とWD5のラウンド上限）。また、未観測の呼び出しサイトや動的ディスパッチのもとでは健全でないため`untyped`へフォールバックする（誤ったナローイングはしない）。
- **WD4 — 健全性のためのフォールバック**。未観測の呼び出しサイト、`send`／動的ディスパッチ、メタプログラミングによる呼び出し元は何も寄与しない（パラメータは`untyped`のまま）。1つの動的呼び出し元が、それ以外は精密な推論を誤ったナローイングへ広げることはない。推論型が過大近似となるのは、呼び出しサイト集合が閉じている場合に限られる。
- **WD5 — 予算＋停止性。実装済み（上限付き、真の収束ではない）**。呼び出しサイトユニオンはワークリスト不動点（worklist fixpoint）である: 各ラウンドで前ラウンドの推論済みパラメータをシードしてプロジェクトを再型付けし（保護スキャンが使うのと同じ`param_inferred_types`の消費経路）、ラウンドごとに1ホップ伝播させる。`DEFAULT_ROUNDS`（3、[ADR-41](../41-inference-budget-design/)の予算。`BodyFixpoint`の慣習）で上限を設け、テーブルが等しくなれば早期停止する──収束は*必須ではない*。なぜならテーブルは際で振動しうる（新たに解決されたレシーバーが、型なしの引数を持つ新たな呼び出しサイトを浮上させうる）し、保護メトリックは有界な近似を許容するからだ。パースはラウンドをまたいでキャッシュされ、再インデックスだけが繰り返される。ADR-57のラン単位の戻り値メモは、延期された`check`ウォークへの配線（そこでは呼び出し先の再型付けが際限なくあってはならない）に向けた将来の再利用先だ。保護のみのパスはそれを必要としない。

## 却下／延期した代替案

| 代替案 | 判定 |
| --- | --- |
| 推論されたパラメータを**宣言済み**として扱う（誤った型の呼び出し元で発火） | **却下** — ADR-5に反する（パラメータは決定により寛容）。基準がそれを禁じる。 |
| 調査の「M3 EXCUSED、追求しない」を維持 | **保護レンズについては置き換え** — *精度*を理由にexcusedされた（リーフスクリプト、呼び出しサイトなし）。保護パイロット＋呼び出しサイトを持つ実アプリがそれを再開させる。 |
| TypeProf完全版の全プログラム抽象解釈器を既定にする | **延期** — Rigorはファイル単位／インクリメンタルである（ADR-46）。呼び出しサイトパスは予算ゲート付き（WD3/WD5）であって、既定の全プログラムワークリストではない。 |
| [ADR-58](../58-ivar-field-typing/)の精度が存在する前にパラメータを推論する | **該当なし — すでに満たされている** — ADR-58の具体的書き込みインスタンス変数型付けは実現済みなので、`@x = param`へ供給される推論済みパラメータは読み取りを即座に鋭くする。 |

## 再評価のトリガー

需要ゲート付き。次の**いずれか**で進める: (a) [ADR-63](../63-type-protection-coverage/)の保護カバレッジが、ユーザープロジェクト全体でM3を最上位の`add_a_type_here`として浮上させ続ける場合（パイロットがその最初のシグナル）。または（b）[ADR-46](../46-incremental-dependency-graph/)のインクリメンタル解析が、ファイル単位モデルの内側でWD3の呼び出しサイトパスを手頃なものにする場合。WD2（本体内の下界）は独立かつ早期に進めてよい──ローカルで偽陽性（FP）安全だからだ。

## 帰結

- **正** — 実アプリケーションに残る最大の保護の穴を閉じる。パイロットで最大の手作業勝利（パラメータ／パラメータ由来インスタンス変数の型付け）を自動化する。基準により精度加算的なので、新たな診断はゼロ。
- **負** — WD3は全プログラム的でファイル単位モデルと緊張関係にある（ゆえに予算ゲート付き）。WD2は構造的下界しか産まない。基準（パラメータ境界で発火しない）が意味するのは、推論が買うのは*保護*であって*境界でより多くのバグを捕まえること*ではない、ということだ──これは設計どおりである。
- **持ち越し** — WD2は安価な第一歩だ（リーフスクリプトにも効く、呼び出しサイト不要）。WD3はアプリ規模のてこだが、インクリメンタリティ／コストの問題を抱える。

## 他のADRとの関係

- **ADR-5** — 整合基準はその直接の適用である: パラメータは寛容なまま保たれ、推論は精度に供されて境界診断には決して供されない。
- **ADR-58** — 兄弟。インスタンス変数フィールド型付けは完了済み。**パラメータ由来**のインスタンス変数は*この*ADRの仕事だ──ADR-58が漸進的としてスコープ外にした、まさにそのM3フロンティアである。
- **ADR-46** — WD3の呼び出しサイトパスの手頃さは、インクリメンタルの筋立てに依存する。
- **ADR-41 / ADR-57** — 予算／停止性と、WD5が再利用するラン単位のメモ基盤。
- **ADR-63** — excusedされていたM3のバケットを再開させた保護パイロット。
