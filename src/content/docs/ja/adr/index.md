このディレクトリにはRigorのアーキテクチャ決定記録（Architecture Decision Records、ADR）が含まれています。各ドキュメントは、重要な設計上の決定、その背景、検討された選択肢、そして結果を記録しています。

## 読み方

- **ADR-0**は基礎ドキュメントです — プロジェクトの中核的な原則とアーキテクチャを知るには、ここから始めてください。
- **ADR-1**から**ADR-3**は型モデル、拡張API、型表現を定義します — アナライザーの概念的な中核です。
- 番号の大きいADRは基礎の上に構築されており、必要に応じて読むことができます。
- 各ADRには**Status**フィールドがあります：`Accepted`、`Proposed`、`Superseded`のいずれかです。実装がまだ進行中のAcceptedなADRは、括弧書きの注記（例: *partially implemented*、*slice N deferred*）を持ちます。

Seam 1 = extract a `DiagnosticOracle` (byte-identical to today) + define the `TestSuiteOracle` interface [ADR-70](70-fused-protection-coverage/) implements; Seam 2 = the type-aware filter becomes one `BiteableSites` strategy, separable so a consumer can pass `AllSites`. No new user surface (the ADR-50 freeze is the CLI/JSON vocabulary, not `Protection::*` class shapes). Rejected: copy the mutator per consumer (maintenance trap); a single both-axes oracle (defeats ADR-70's gradual short-circuit); a public plugin API now (deferred to ADR-71). Co-land with ADR-70's first dynamic consumer so the interface is exercised, not speculative. Grounding: the 2026-06-17 strategy note |
| ADR-70 | [Fused static∪dynamic protection coverage](70-fused-protection-coverage/) | **Accepted — implemented 2026-06-17 (`coverage --protection --mutation --with-tests`), co-landed with ADR-69, validated on faraday / liquid / mail.** Adds an optional **dynamic overlay** to `coverage --protection --mutation`: for each mutant the *type* checker fails to kill, ask whether a *test* kills it (the ADR-69 `TestSuiteOracle`, runner hook `--test-command`, run with a stripped bundler env), and classify each site by **both** axes — type-protected / test-protected / **unprotected** (the conceptual *doubly-protected* bucket collapses into type-protected: a type-killed mutant never reaches the suite — the gradual short-circuit). `--include-dynamic` (ADR-69 Seam 2) extends it to `Dynamic`-receiver sites, completing the map (liquid `lexer.rb` 76 → 115 sites, 38 test-protected + 2 real gaps). The artifact is the fusion: a line is truly unprotected only when **neither** a type nor a test guards it, and the report names the **cheaper missing axis** ("add a type" vs "add a test"). No tool fuses static type-protection and dynamic test-protection into one map. Criterion (extends ADR-63's framing): always effectiveness / where-to-add-protection, never raw survival — and the payload is the **attribution** (cheapest missing axis), not a single number; a site is unprotected only when both axes miss. Gradual short-circuit: a type-killed mutant never reaches the suite (type = first line, tests = second), so cost is proportional to the protection hole. WD1 survivor-scoped, opt-in, changed-files default; WD2 the test oracle is an **adjudicated candidate, not a verdict** ([ADR-59](59-spec-assertions-are-not-signatures/): pending/tag/flaky tests are unknowable — a survivor prioritizes attention, it does not certify a gap); WD3 consume ADR-69's seam, `DiagnosticOracle` path byte-identical; WD4 new flag + JSON keys frozen under ADR-50. **Selection is the suite's job, NEVER the dependency graph** (a type-read graph ≠ the runtime call-graph → skips the killing test → a **false survivor** = an FP, the line `feedback_false_positive_discipline` won't cross). Rejected: run the suite for every mutant (no short-circuit); a single fused %; graph-based test selection; a new `rigor teeth` command; treating a test-survivor as a proven gap. Grounding: the 2026-06-17 strategy note (judgement step 2) |
