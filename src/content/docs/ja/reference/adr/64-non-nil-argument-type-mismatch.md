Status: **Accepted —— 非nilチャネルは構築・ゲートされた**。 `nil`チャネルは本サイクルの早期にFP安全に出荷された（`a2775c7b`が多重オーバーロード、`0afc65ea`がインターフェイスエイリアス;CRuby `lib/`に対する[ADR-62](../62-mutation-testing-teeth-measurement/)の歯（teeth）スイープ）。非nil（`type_swap`）チャネル —— Rubyの`coerce`プロトコルの背後に先送りされていた、残存する最大の単一の偽陰性クラスタ —— は今や**多重オーバーロード**メソッドに対して出荷される（下記のWD1+WD2+WD3）: coerceディスパッチ演算子は除外され、受理は1つの具体的なRBS既知の引数クラスに対しRBSパラメータ型（`param_accepts_arg_class?`）で判定される。それは自らに課したゲートを達成した —— **31の`rigor-survey`プロジェクト（Mastodonの`app` + `lib`を含む）にわたる新規発火ゼロ、かつミューテーションの歯`type_swap`のキル数が上昇**（kramdown 5 → 9、liquid 0 → 7）。既存の単一オーバーロードのパラメータごと受理チェックは不変;それがカバーする比較演算子（`Integer#< : (Numeric)`は単一オーバーロード）は従来の挙動を保つ。

根拠: [`docs/notes/20260613-mutation-teeth-harness.md`](../../notes/20260613-mutation-teeth-harness/)（スイープ + nilチャネルの着地）、[ADR-62](../62-mutation-testing-teeth-measurement/)。

## コンテキスト

`call.argument-type-mismatch`は、呼び出し引数の型がパラメータの型と非互換のとき発火する。本サイクルはこれを**nilチャネル** —— すべてのオーバーロードの対応する位置パラメータに拒否される純粋な`nil`引数 —— へ拡張し、nil受理をRBSパラメータ型で判定するので多重オーバーロードメソッドとインターフェイスエイリアスパラメータを見通す（`check_rules.rb`の`param_admits_nil?`）。これはFP安全に着地した: 12の`rigor-survey`プロジェクトにわたる新規発火ゼロ。

ADR-62のミューテーションスイープは、**非nilチャネル** —— 誤った*型*の引数（`type_swap`: `String`が期待される場所での`Integer`、…） —— を、最大の残存偽陰性クラスタとして残す（約900ミュータント、対するnilチャネルは今や約520がキル済み）。ルールを非nil引数へ拡張すべきかが、この判断だ。

`nil`が回避したブロッカーは、**Rubyの`coerce`プロトコル**である。`5 + Money.new`はランタイムでは有効だ —— 引数が`Numeric`でないとき`Integer#+`は`Money#coerce(5)`を呼ぶ —— RBSのどの`Integer#+`オーバーロードも`Money`を列挙していないにもかかわらず。素朴な「どのオーバーロードも引数を受け付けないとき発火する」ルール（決して型強制しない`nil`には健全）は、**あらゆる型強制可能なユーザー型で偽陽性を出す**。それこそが`nil`がFP安全のコアだった理由である。

## 決定

FP安全な形を記録し、**実装は需要ゲート付きとする**。

**基準（再利用可能なルール）**。非nil引数は、**ランタイムの受理が型だけから決定可能**な箇所 —— `coerce`も変換プロトコルもそれを救えない箇所 —— でのみ報告してよい:

- **coerceディスパッチ演算子を除外する**。二項算術 / 順序演算子（`+ - * / % ** & | ^ << >> < > <= >=`と`Comparable`ミックスインの`<=>`由来の形式）は`coerce` / `<=>`を介してディスパッチする;それらへの非`Numeric`引数は*決して*静的に反駁できない、ユーザー型が`coerce`を定義し得るからだ。非nilの引数チェックはそれらには適用されない。（`nil`はそこでもカバーされたまま —— 型強制できないので。）
- **残りの（非演算子）メソッドについては、`param_admits_nil?`を`param_accepts_arg_class?`へ一般化する** —— 引数の具体的なクラスは、トランスレータが劣化させるRBSインターフェイスを解決した上で、パラメータを満たすか？（`Integer`は`_ToStr`を実装するか？ しない → `"a".sub(0, …)`は本物の不一致だ。）これはnil作業のエイリアス / インターフェイス解決（`RbsLoader#expand_type_alias`、`interface_method_names`）を再利用し、「NilClassが」ではなく「引数クラスがインターフェイスのメソッドに応答するか」を問う。
- **構築上保守的**、nilチャネルと同様に: 決定不能なパラメータ（型変数、未解決のインターフェイス、`untyped`）は受け付け、ルールは積極的に反駁された具体的な引数クラスでのみ発火する。

**ゲート（いつ構築するか） —— 達成**。限界的な歯が追加のFPサーフェスを正当化するときだけ出荷する: 非演算子の非nilチャネルが**新規コーパス偽陽性ゼロで本物の歯を生む**ことを示す`rigor-survey`コーパスパス —— nilチャネルがクリアしたバー。構築（多重オーバーロードチャネル）はそれをクリアした: 31の`rigor-survey`プロジェクト（lib持ちのライブラリ、アルゴリズムコーパス、そしてMastodonの`app` + `lib`の目玉FPベンチマーク）にわたるbefore/after差分が**新規の`call.argument-type-mismatch`発火ゼロ**を見出し、一方`type_swap`ミューテーションスイープはkramdownで5 → 9キル、liquidで0 → 7キルへ上昇した。高FPリスクのサーフェス（単一のRBS既知の引数クラスに対する有界なインターフェイス解決）は実コードでは沈黙を保った、正しいプログラムは正しく型付けされた引数を渡すからだ;歯はミューテーション下でのみ現れる。

## ワーキングデシジョン

- **WD1 —— coerce演算子の除外はヒューリスティックではなく固定の許可リスト**。演算子は列挙可能な集合だ;`coerce`定義を検出するのではなく、既存の`UNIVERSAL_EQUALITY_METHODS`の免除（`==` / `<=>`ファミリーは既に免除）のように`COERCE_DISPATCH_METHODS`集合でモデル化する。この拡張は*非nil*ケースにのみ適用される。
- **WD2 —— `param_accepts_arg_class?`はnil述語の一般化であり、新しい`Acceptance`パスではない**。 RBSパラメータ型に対するcheck-rulesレイヤーに留める（インターフェイスを意図的に漸進的へ劣化させる`Inference::Acceptance`ではなく）。引数側は「クラス`C`がインターフェイス`I`を実装する」になる —— `nil_class_has_method?`が`NilClass`に対して行うように`C`のメソッド集合をルックアップする。
- **WD3 —— 単一の具体的な引数クラスに限定する**。 nilチャネルの純粋nil限定と同様に、引数が1つの具体的なクラスに型付けされる（ユニオンでも漸進的でもない）ときだけ発火する;ユニオン引数はユニオンレシーバーのストーリーを映し、先送りのまま。

## 却下／先送りした代替案

| 代替案 | 裁定 |
| --- | --- |
| すべての非nil引数に対する素朴な「どのオーバーロードも引数を受け付けない」 | **却下** —— あらゆる型強制可能なユーザー型で偽陽性を出す（`5 + Money.new`）。coerceの障壁こそが`nil`が安全なコアだった理由全部だ。 |
| 演算子チェックを保つために`coerce`をモデル化する（どの型がそれを定義するか追跡する） | **却下** —— 有界でない（任意のユーザー型が`coerce`を定義し得る）;演算子除外が安価で健全な近似だ。 |
| 引数クラスに`coerce`メソッドがあるか調べてcoerceを検出する | **先送り** —— 除外を狭め得る（引数クラスが`coerce`を証明可能に欠くとき発火）が、既にゲートされたチャネルにサーフェスを追加する;演算子-引数の歯が要求されたときだけ再訪する。 |
| 非演算子チャネルを今構築する | **先送り** —— nilチャネルを越える実証されたコーパスの歯がない;上記のトリガーで出荷する。 |

## 帰結

- **正** —— coerce基準（再利用可能な「型だけから決定可能」ルール）を記録するので、次の実装者がFPの罠を再導出しなくて済む;nilチャネルの`param_admits_nil?`が一般化する実証済みの基盤だ。
- **負** —— 最大の歯クラスタは開いたまま;その高確信部分（`nil`）が既に着地し、残りはcoerce拘束または低収率なので受容する。
- **持ち越し** —— `expand_type_alias`での名前空間化された型エイリアス解決（nilチャネルが表面化させた`MatchData#[]` / `::MatchData::capture`のギャップ、エイリアスがトップレベルでない箇所）は、チャネルが構築される場合にWD2のインターフェイス作業に同行する。
