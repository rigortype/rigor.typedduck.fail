**ステータス:**作業メモ、設計上の確約なし。2026-06-05時点のADRコーパス（このコミット時点）に対し、[ADR-49](../../adr/49-adr-authoring-guidelines/)の執筆ルーブリック（rubric）を適用した時点測定。スコアは**ルーブリックに照らした一読者の判断**であり、審査基準ではありません。目的は各ADRのスコアを下げている1〜2軸を特定し、ADR-49をSKILLに確定させる前に実データで検証することです。仕様書とADR自体は拘束力を持ちますが、このノートは持ちません。

読了順のコンパニオン: [ADR-49](../../adr/49-adr-authoring-guidelines/)でアーキタイプ、stakesタグ、以下で採点する8軸を定義しています。

## 方法

各ADRに**アーキタイプ**（Deliberative / Mechanical-policy / Evaluation-proposal）と**stakes**（low / mid / high、可逆性×影響範囲×FPエンベロープで決定）をタグ付けし、8軸で採点します。各軸は`0`/`1`/`2`、またはアーキタイプ上免除される軸は`–`（分母から除外し、`0`にはしない）:

| キー | 軸 |
| --- | --- |
| **I** | Intent & purpose（意図と目的） |
| **C** | Decision criterion（決定基準） |
| **A** | Actionability & code-anchoring（実行可能性とコードアンカー） |
| **G** | Constraints & guardrails（制約とガードレール） |
| **V** | Verifiability & evidence（検証可能性と根拠） |
| **S** | Status & progress fidelity（ステータスと進捗の正確さ） |
| **X** | Connectedness（接続性） |
| **E** | Economy / proportionality（経済性／比例性、双方向） |

`%` = `sum / (2 × scored-axes)`。ADR-49に従い、低スコアは「どの軸が引き下げているか」を示すものであり、「ADRが悪い」意味ではありません。低スコアの多くは適切な経済性や経年によるものであり、欠陥ではありません。

## 所見

### 1. 監査の主要な出力は空 — それが安心できる結果

ADR-49の最も価値ある警告シグナルは**high-stakes × 意図が薄い**（根拠なしに記録された重要な決定）です。**コーパスにそのような例はありませんでした**。 `high`-stakesのすべてのADR（0, 1, 2, 3, 4, 5, 10, 15, 16, 17, 20, 24, 26, 31, 34, 35, 37, 39, 43, 45, 46）は、IntentとCriterionの**両方**で`2`を獲得しています。ルーブリックのアラームは一度も発動しませんでした。これは「まさにそのギャップを検出する」ために設計されたルーブリックにとって、コーパスがテストをパスしたことを意味し、ルーブリックが何も見つけられなかったわけではありません。

### 2. 経済性が唯一の繰り返し引き下げ要因 — ADR-49の目的を裏付ける

コーパス全体で最も失点が多い軸は**軸8（経済性）**です: 50件中約22件がこの点を落としています。コーパスは**過剰情報**に傾いており、情報不足のADRは1件もありません。実装するには薄すぎるADRは存在しません。これはADR-49の双方向経済性軸が名付けようとしている非対称性そのものであり、「stakesに対する簡潔さ」をADR-49が押すレバーにしたことの正当性を検証しています。

過剰情報の最も明確な例（stakesを考慮しても長さが不釣り合い）:

- **ADR-22**（960行） — ベースラインメカニズムで、WD9の代替案分析だけで他のADR全体より長くなっています。決定内容は適切ですが、散文は`mid`-stakesのオンボーディング機能が必要とする量の約2倍です。
- **ADR-16**（1221行）と**ADR-1**（850行） — `high`-stakesなので長さはより擁護できますが、両方ともリンク先のノートに移動できる大きな参照/比較テーブルを含んでいます。
- **ADR-21 / ADR-30 / ADR-42** — *evaluation*アーキタイプ、`mid`/`low`のstakesにもかかわらず長い。「defer / demand-gate this」という決定が完全熟考の長さで書かれているのが、low-stakes側における比例性の問題です。

経済性を適切に守っている反例（キャリブレーションのターゲット）: **ADR-33**（MCP、約120行、簡潔かつ完全）、**ADR-38**（追加イニシャライザ、密度高く適切）、**ADR-44**（アロケーションチャーン、密度高くシグナルが多い）、**ADR-40**（configデフォルト — 小さな機械的変更を小さく書いた例）。

### 3. 低スコアはアーキタイプ/経年によるもので欠陥ではない — 過剰減点防止ルールが機能している

テーブルの下位は**弱いADRではありません**:

- **ADR-0（63 %）** — コード前段階の基盤;コードアンカーや代替案基準がまだ存在しなかったため記録されていない。低いのは正当で、欠陥ではない。
- **ADR-12（69 %）、ADR-7（71 %）、ADR-33（75 %）** — IntentがもともとMUSTではないmechanical/bundleアーキタイプ。ADR-7のIntentは`–`（N/A、14点満点で採点）;免除なしならはるかに低く読めるところ、N/Aメカニズムが機能している。
- **ADR-27（75 %）** — 一部実装中の配布ポリシー; Status/Actionの点は実際の進行中状態を反映しており、これは正直な記録であり文書化のギャップではない。

本当に問題となる理由（実際の推論ギャップ）でスコアが低いADRはありませんでした。これはADR-49の免除が誤った失敗を生み出さないことの根拠です。

### 4. コーパスの本質的な強みは3軸に集中している

**Criterion（C）**、**Guardrails（G）**、**Connectedness（X）**はほぼ全件で`2`。コーパスは識別可能な決定ルール（ADR-43の許可リスト基準、ADR-35の方向非対称性、ADR-15のfork-vs-Ractorテーブル）の記録、FP/健全性エンベロープの明示（プロジェクトの最重要価値であり、ほぼすべてのガードセクションに現れる）、クロスリンクの3点が際立っています。これらはSKILLがデフォルトで保持すべきハウスストレングスであり、再教育を要するものではありません。

### 5. 検証可能性（V）がアーキタイプ/経年のシグナルとして最も明確

Vは明確に分かれます: **最近の熟考型** ADRは具体的な数値を持ち（ADR-43のゼロnet-FP、ADR-45の11.6→1.8s、ADR-35の160→35 FP、ADR-24の467→15、ADR-15のバグ#22075、ADR-44のGC −29 %）、`2`を獲得; **古いまたは設計のみの** ADR（5, 6, 9, 13, 25, 28）は論拠はあるが未計測で`1`; **evaluation/proposal** ADRは調査根拠を適切に持つが出荷済み計測はない（ADR-41は際立った例 — *提案として*計測主導）。これはADR-49の「提案にはevidenceサブパートN/A」免除が最も重要な軸であり、データはこれを維持する根拠を支持しています。

### 6. ステータス/本文の整合性（S）が均一に高い — 乖離なし

すべてのADRの本文がそのStatusラインと一致していました。最近のADRがlandedステータスの密度の高い冒頭段落を持つ習慣（ADR-46、ADR-47、ADR-24）は、レジュメブックマークとしても機能します — `S`は高いが、小さな`E`コストを伴います（ステータスの羅列は経済性を少し削る）。これはSKILLが抑制するのではなく承認すべき意図的なトレードオフとして記録する価値があります。

## 模範例とアンチパターン（ADR-49のキャリブレーションセット）

- **総合最優秀（100 %）: ADR-26, 43, 44, 45, 46**。すべて熟考型、すべて最近、すべて再利用可能な決定基準 + 計測済みFP/パフォーマンス結果 + 密度の高い本文を持つ。ADR-43（基準）とADR-46（健全性 + 進捗）はすでにADR-49の命名アンカーです; **ADR-45**（却下された非健全設計から健全設計へのアーク）と**ADR-26**（最初の試みを取り消した → 診断レイヤー修正）が最も強力な追加例 — 両方とも「誤った選択を記録する」ことを教えています。
- **最優秀mechanical: ADR-40 / ADR-38 / ADR-33** — 小さな変更を小さく書き、却下された代替案テーブルでCriterion `2`を獲得しながら肥大化していない。ADR-49はすでにADR-40を引用しています; ADR-33は最も純粋な「簡潔なMCP型」例です。
- **経済性のアンチパターン: ADR-22** — コーパスで最も明確な「正しい決定、過剰記述」の例。経済性軸の負の模範例として適切です（*間違い*ではなく*長い*）。SKILLは`E=1`の見た目を示すためにこれを使用でき、ADRの品質が低いことを示唆せずに済みます。

## ADR-49への示唆（およびキュー待ちのSKILL）

1. **ルーブリックはコーパスとの接触に耐えた**。そのアラーム（high-stakes × 薄い意図）は正しく沈黙を保ち、免除は機械的/基盤ADRを正しく救済し、一つの有効なレバー（経済性）は実際の、一貫したコーパストレンドを特定しました。無効な軸はなく、偽の失敗を生んだ免除もありませんでした。ADR-49はSKILLに確定させる準備が整っています。
2. **経済性は前面に出すべきレバー**。過剰情報がコーパスの唯一の系統的な傾向であるため、起草SKILLは*stakesに対する長さ*を最も目立つプロンプトにすべきです — 例えば、evaluationやlow-stakes ADRが〜ADR-33の長さを超えた場合、「このコンテンツをリンク先のノートに移動するか？」チェックを促すべきです。参照テーブル抽出パターン（ADR-1 / ADR-16が候補）が具体的な解決策です。
3. **ADR-49への小さな改善提案（ここでは未実施）:**「密度の高いlandedステータス段落」は、レジュメブックマーク価値を得るために繰り返し行われる*意図的な*経済性の削減です（所見6）。ADR-49の経済性軸は現在これを軽微な`E`コストとして読んでいます; SKILLはこれを欠陥ではなく承認されたトレードオフとして明示すべきで、著者が経済性ポイントを追いかけるために重要なステータスブロックを削らないよう対処すべきです。
4. **この監査の根拠に基づく個別ADRへの修正編集は必要ない**。すべての「引き下げ要因」は、経年（ADR-0）、アーキタイプ（ADR-7/12/33）、正直な進行中状態（ADR-27）、またはhigh-stakes ADRの比例的な長さのいずれかです。経済性の外れ値（特にADR-22）は、将来*trim-and-link*パスが望まれる場合の候補ですが、どれも正確性や十分性のギャップではありません。

## 注意事項

- スコアは単一パスにおける一読者のキャリブレーションされた判断です; deliberative↔mechanicalの境界にあるアーキタイプ/stakesタグ（ADR-19、ADR-31等）は議論の余地があり、1〜2点程度変動します。
- この監査は文書を採点したものであり、実装ではありません — ADRはActionability（注入ポイントを名指している）で`2`を獲得できますが、その名指されたサイトはその後移動している可能性があります;このノートは標準の「実行前に名指されたファイル/シンボルが現存することを確認」という注意事項を継承します。
- 経済性の判断は最も再現性の低い軸です（長さ対stakesは本質的に主観的）;これらはSKILLのプロンプトデザインへのシグナルとして提示しており、権威ある評価ではありません。
